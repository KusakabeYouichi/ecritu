第7章設定リファレンス

écritu の設定は、ホーム画面の「écritu」アプリ(コンテナーアプリ)を開いて行います。この章では、アプリに並ぶすべての設定項目を、画面の表示順どおりに 1 つずつ説明します。

設定はすぐ反映されます:アプリで設定を変更すると、その場でキーボードに通知されます。キーボードを使っているアプリに戻るだけで反映され、再起動などは不要です。唯一の例外は「フリック方向」で、次にキーボードを表示したときに切り替わります。

設定画面の見かた

アプリを開くと、アイコンの下に édition n°〜 という小さな番号が表示されています。これは écritu のバージョン(エディション)番号で、ごく小さく表示されていますが、長押しすると拡大表示されて読み取れます。不具合の報告などでバージョンを尋ねられたときはここを確認してください。

その下に設定カードが 1 列に並びます。以降の節は、このカードの並び順そのままです。

配列と入力方式

フリック方向

選択肢Apple / écritu
既定écritu

フリックしたときにどの方向でどの文字が入るか(方向の割り当て方式)を切り替えます。écritu 方向は本キーボード独自の割り当て、Apple 方向は iOS 標準キーボードと同じ割り当てです。iOS 標準に慣れた指をそのまま使いたい方は Apple を選んでください。切り替えは次回のキーボード表示時に反映されます。両方式の全キー対照は第8章 配列リファレンス(かな)を参照してください。

かな修飾

選択肢前置修飾 / 後置修飾
既定前置修飾

濁点・半濁点・拗音/促音(小文字)の入力方式を切り替えます。前置修飾は「修飾キーを先に選んでから文字を打つ」方式、後置修飾は iOS 標準と同じく「文字を打ってから修飾キーで変化させる」方式です。それぞれの操作手順は第8章で図解しています。後置修飾を選んだ場合は、後述の「タップ(後置修飾、未確定なし)」「後置修飾キーのフリックで濁点・半濁点」の 2 項目も関係します。

かな配列

選択肢5×2 / 3×3+わ
既定5×2

かなモードのキー配列を切り替えます。5×2 は écritu 標準の配列(あ〜な行とは〜わ行の 2 段構成)、3×3+わ は Apple 標準の日本語配列に合わせて各段 5 ボタン(かな 3 + 機能 2)で表示する配列です。両配列の全体図は第8章にあります。

ラテン文字候補ペイン (horizontal)

候補ペインを使うチェックボックス(既定: オン)
並び順『候補』『ラテン文字』をドラグで左右入替(既定: 候補が左)

iPhone を横向きにしてラテン文字(英字)を入力するときに、キーの横へ候補ペイン(候補一覧の縦領域)を表示するかどうかの設定です。オンのときは『ラテン文字』『候補』の 2 枚のカードをドラグして、どちらを左に置くかを入れ替えられます。チェックを外している間は左右配置を変更できません。横画面のレイアウト全体は第8章(続) 配列リファレンス(横画面)を参照してください。

数字ペイン配列 (horizontal)

並び順『数字』『記号』をドラグで左右入替(既定: 数字が左)

横向きで数字 3×3 入力を使うときの、数字パッドと記号パネルの左右配置です。かな入力モードの候補配置には影響しません。詳細は第8章(続) 横画面へ。

ラテン文字配列

選択肢azerty / qwerty / 3×3
既定azerty

abc モード(英字入力)の配列を切り替えます。qwerty / azerty はフルキーボード型で、文字キーを長押しするとアクセント付き文字(é、à など)を入力できます。3×3 はかなと同じフリック型の英字配列です。各配列の全キーは第8章(続) 配列リファレンス(英字・数字)を参照してください。

数字配列

選択肢calculette / téléphone / clavier
既定calculette

123 モードの数字キー配列を切り替えます。calculette(電卓)は上段が 7-8-9、téléphone(電話)は上段が 1-2-3 です。clavier は AZERTY 風の数字+記号配列で、shift で 2 種類の記号セットを切り替えられます。ただし clavier は縦画面のみの対応で、横画面では自動的に calculette が使われます。詳細は第8章(続) 英字・数字へ。

基本記号の並び順

選択肢ASCII / EBCDIC / ANSI
既定ASCII

記号モードの『基本記号』カテゴリに並ぶ記号の順序を、文字コード規格の順に切り替えます。通常は既定の ASCII 順のままで問題ありません。記号パネルの使いかたは第5章を参照してください。

色とガイド表示

アクセントカラー

選択肢tuilé(チュイレ) / émeraude(エメロード)
既定émeraude

キーを押したときのハイライトなど、アクセント色を切り替えます。tuilé は瓦の色(赤茶)、émeraude は宝石の色(緑)です。

テーマカラー

選択肢bleu ciel brumeux(かすみがかった空の青) / rose sakura poudré(桜の淡い桃色)
既定bleu ciel brumeux

キーボード背景のグラデーションを切り替えます。設定画面の左に表示される色見本が、そのまま実際の背景色です。ライト/ダークモードにはどちらのテーマも自動で追従します。

ガイド文字表示

対象かな入力 / ラテン文字入力 / 数字入力 / 濁点・半濁点・小文字キー(それぞれ個別に設定)
選択肢オフ / 4方向 / 下
既定すべて 4方向

各キーのまわりに小さく表示される「フリックすると何が入るか」のガイド文字の表示方法を、入力モードごとに選べます。4方向はキーの上下左右にガイドを配置、下はメイン文字の下にガイド文字を横並びで表示、オフは非表示です。濁点・半濁点・小文字キー(修飾キー)のガイドは、入力モードの設定とは独立して適用されます。なお、ラテン文字配列が 3×3 以外(qwerty / azerty)のときは、ラテン文字入力のガイドは表示されず、この欄は選択できません。

キーの反応と自動確定

削除キーリピート

リピート開始までの時間0.1〜0.8 秒(既定: 0.5 秒)
リピート速度(間隔)0.05〜0.2 秒(既定: 0.1 秒)

削除キー(⌫)を押し続けたときの連続削除の設定です。1 回目の押下でまず 1 文字削除され、「開始までの時間」が過ぎると「間隔」ごとに連続削除されます。スライダーの値が既定値のときは「デフォルト」バッジが表示され、変更すると「デフォルトに戻す」ボタンで戻せます。

自動確定(アイドル)

入力が止まったら未確定を自動確定オン / オフ(既定: オフ)
確定までの待ち時間0.3〜5.0 秒(既定: 1.2 秒。トグルがオンのとき表示)

先行する確定文字があると、未確定(下線付き)の文字はメッセージ送信時に切り捨てられます。これは iOS の拡張キーボード共通の制約です(第9章 困ったとき参照)。この設定をオンにすると、入力が設定した時間だけ止まったときに未確定部分を自動で確定し、送信に乗るようにします。行頭から全部が未確定のときは元々送信に乗るため、自動確定の対象外です。

キー割り当て

かな左下キー割り当て

対象タップ / 右フリック / 上フリック(それぞれ個別に設定)
選択肢絵文字 (☺︎) / 顔文字 (^_^) / 記号 (⌘)
既定タップ=絵文字、右フリック=顔文字、上フリック=記号

かな入力モードの左下キーから、絵文字・顔文字・記号の各パネル(第5章)のどれをどの操作で開くかを設定します。同じ機能を複数の方向に割り当てることもできます。キーに表示される記号(☺︎ / ^_^ / ⌘)は、タップに割り当てた機能のものになります。

タップ (後置修飾、未確定なし)

選択肢顔文字入力モードに一時切り替え / 絵文字入力モードに一時切り替え / 記号入力モードに一時切り替え
既定顔文字入力モードに一時切り替え
切替時に開くカテゴリー切り替え先ごとに初期カテゴリーを選択(既定: 顔文字=基本、絵文字=ひと、記号=基本記号)

かな修飾で「後置修飾」を選んでいるときの専用設定です。未確定文字が何もない状態で修飾キー(゛゜キー)をタップしたときに、どの入力モードへ一時的に切り替えるか、そして切り替え先で最初にどのカテゴリーを開くかを指定します。切り替え先で 1 つ確定すると、自動的にかな入力モードへ戻ります。

後置修飾キーのフリックで濁点・半濁点

選択肢オン / オフ
既定オン

こちらも後置修飾のときの設定です。オンの場合、後置修飾キーを上フリックで濁点(゛)、右フリックで半濁点(゜)を強制入力します。オフにすると上/右フリックは中央タップと同じ扱いになり、フリックのつもりがなかったのに「つ→づ」になってしまう誤操作を抑止できます(オフでも、タップ 2 回で「つ→っ→づ」の順に変化させることは引き続き可能です)。

変換候補のふるまい

句読点入力時の自動確定候補

選択肢先頭の変換候補 / 未変換かな
既定先頭の変換候補

未確定の文字がある状態で句読点や記号を入力すると、未確定部分が自動確定されます(第3章)。そのときにどの候補で確定するかの設定です。「先頭の変換候補」は候補バーの先頭に出ている変換結果、「未変換かな」は入力したひらがなをそのまま確定します。なお、確定キーによる確定はこの設定に関わらず常に未変換かなを確定します。

かな漢字候補モード

選択肢normalisé / surface / les deux
既定surface

システム辞書から候補を採用する基準を切り替えます。surface は辞書の表記(表層形)どおりの候補、normalisé は正規化された表記の候補、les deux は両方を採用します。通常は既定の surface のままで問題ありません。

旧仮名遣い候補

選択肢オン / オフ
既定オフ(現代仮名遣いのみ)

「かえる→変へる」「かんがえる→考へる」のような歴史的仮名遣い表記の候補を、変換結果に含めるかどうかを切り替えます。

emojis & les émoticônes(絵文字・顔文字候補)

emoji 😀オン / オフ(既定: オン)
émoticône (^_^)オン / オフ(既定: オン)

かな漢字変換の候補に絵文字/顔文字を含めるかを、それぞれ切り替えます。オンのとき、たとえば「ねこ」の変換候補に 🐱 などが並びます。読みからの絵文字変換のしくみは第5章を参照してください。

iOSの連絡先の姓、名、会社名

選択肢連絡先候補を使わない / 一致した名前だけ候補にする / 姓/名一致でフルネームも候補にする
既定一致した名前だけ候補にする

iOS の連絡先に登録された名前を変換候補に使う設定です。「一致した名前だけ」は姓・名・ニックネームなど読みが一致した項目のみを表示し、「フルネームも候補にする」は姓または名が一致したときにフルネーム全体も候補に加えます。オフにすると連絡先由来の候補は表示されません。オフ以外を選ぶと、初回に連絡先へのアクセス許可を求められます。詳細は第6章 語彙と学習へ。

iOSのユーザ辞書の単語

選択肢使わない / 使う
既定使う

iOS 本体の [設定] > [一般] > [キーボード] > [ユーザ辞書] に登録された単語を、écritu の変換候補として使うかどうかを切り替えます。écritu 自身に単語を登録したい場合は、次節の追加語彙を使ってください(第6章)。

語彙の管理(概要)

ここからの 6 つのカードは écritu の語彙(単語リスト)の管理です。使いかたの詳細は第6章 語彙と学習で説明しているため、ここでは各カードの役割だけまとめます。

追加語彙

手動で登録する単語(候補+よみ)のリストです。登録・編集・削除のほか、「初期語彙再投入」「全削除」ボタンがあります。登録した単語はキーボード拡張と共有され、候補の優先順位に反映されます。→ 第6章

学習語彙

確定操作から自動登録された候補のリストです。個別削除のほか「学習語彙全削除」(リストだけ消す)と「学習リセット」(学習スコアごと消す、より完全な初期化)があります。キーボードの「フルアクセスを許可」がオフだと学習は保存されません(第2章)。→ 第6章

抑制語彙

候補に出したくない単語を「よみ+単語」の組み合わせで登録するリストです。同じ単語でも別の読みの候補には影響しません。→ 第6章

ショートカット語彙

顔文字パネルの先頭側に表示される定型文のリストです。よみは ☻ で固定され、固定顔文字とは区切り線で分離されます。→ 第6章

第1語彙・第2語彙(読み取り専用)

écritu に内蔵されたシステム辞書(Dictionnaire système premier / secondaire)の中身を閲覧するカードです。件数が多いため最初は読み込まれておらず、「第1語彙を読み込む」「第2語彙を読み込む」ボタンで一覧を表示します。読み取り専用のため、追加や削除はできません。

その他

有効化手順

キーボードを有効化する 3 ステップ(新しいキーボードを追加 → 有効化 → 地球儀キーで切り替え)のメモが表示されます。画面つきの詳しい手順は第2章 導入を参照してください。

オープンソースライセンス

「ライセンス表示を開く」から、écritu が利用している SudachiDict 関連などのオープンソースライセンス文書を確認できます。

キーボード診断ログ

キーボード拡張の動作記録(診断ログ)を確認するカードです。稼働中/停止の状態、fail-safe プロファイル、最終ハートビート、最終セッション ID、最終イベント、インストール識別子が表示され、ボタンで操作します。

更新ログと状態を読み直します。
コピーログ全文をクリップボードにコピーします(不具合報告に添付するときに便利です)。
ログクリア確認のうえ、保存済みの診断ログを削除します。

通常の利用でこのカードを操作する必要はありません。キーボードの調子がおかしいときの確認方法は第9章 困ったときにまとめています。
