# 誤検知回避メモ

このメモは、R01〜R12 の再検査時に確認できた誤検知回避条件を簡潔に整理したものです。
SKILL.md 本体の補助資料として扱い、検査ロジックを見直すときの前提にします。
実際の再検査手順は `./RESCAN_GUIDE.md` を参照してください。

## 共通前提

- 対象はプログラムのソースを含むテキストファイルです。
- `build/`、`build-device/`、`build-device-agent/`、`third_party/sudachidict/`、`écritu.xcodeproj/xcuserdata/` は対象外です。
- Markdown のコードブロックは原則として実コード検査の対象外です。
- コメントと文字列リテラルは、実コード規則の判定対象から外します。

## ルール別メモ

### R07

- 連続空行の判定は、コード系拡張子だけを対象にします。
- Markdown 本文の段落整形は対象外です。

### R09

- Markdown コードブロック内部は対象外です。
- 括弧を伴わない空の型定義など、言語仕様上必要な書式は違反扱いしません。

### R10

- URL やプロトコル文字列は対象外です。
- 区切り線としての `//` 単独行は対象外です。

### R11

- 実コードの括弧だけを対象にします。
- コメント、文字列リテラル、ドキュメント用コードブロックは対象外です。

### R12

- カンマ直後に実コードが続く場合だけ、半角空白1つを要求します。
- 行末コメントしか続かない場合は、空白や `tab` による位置合わせを許容します。
